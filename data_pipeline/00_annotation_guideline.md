# 网易云风格乐评 · 数据标注规范 v1.0

> 本文是「乐评微调数据集」的**标注指南（Annotation Guideline）**。
> 所有标注员必须严格按本规范操作，保证标注数据的一致性、质量与可复现性。
> 对应岗位职责：*深入理解大模型训练对数据标注的规范和要求，严格按照标注指南进行操作*。

---

## 0. 背景与目标

我们要为「网易云风格乐评生成」大模型准备**微调训练数据（SFT）**。
原始语料来自音乐评论区，噪声极大（灌水、超短、表情、广告、跑题、繁体、火星文……）。
标注员的任务：**从原始评论中筛选、清洗、并标注出「高质量乐评」**，
作为模型学习"网易云文风"的高质量燃料。

一句话标准：**模型学了这条数据，会不会变得更会写打动人的乐评？** 会 → 保留；不会 → 丢弃。

---

## 1. 什么是「合格的乐评」—— 四个质量维度

每条候选评论，按以下 4 个维度各打 **0~3 分**（0=完全没有，3=非常突出）：

| 维度 | 含义 | 3 分示例 | 0 分示例 |
|---|---|---|---|
| **画面感** | 有具体场景 / 细节 / 时间地点，能"看见" | "凌晨三点的客厅，只有电脑屏幕亮着" | "好听" |
| **共情力** | 戳中普遍情绪（孤独/成长/怀念/遗憾） | "有些告别，就是最后一面" | "999+" |
| **故事感** | 有人物 / 情节 / 时间线的小片段 | "每天坐同一班地铁，总会遇到那个戴耳机的女生" | "前排" |
| **音乐关联** | 和歌曲 / 歌手 / 旋律有呼应 | "这首歌前奏一响，那些人和事全回来了" | "今天中午吃的螺蛳粉真香" |

### 合格线（total = 四维之和，满分 12）

| 总分 | 等级 | 处理 |
|---|---|---|
| **≥ 8** | 优质（A） | **保留**，进入训练集 |
| 5 ~ 7 | 中等（B） | 保留，但需**改写增强**后入库 |
| ≤ 4 | 不合格（C） | **丢弃** |

> 训练集只收 A 级（和改写达标的 B 级）。宁缺毋滥 —— 100 条精标 > 1000 条平庸。

---

## 2. 必须直接打回 / 丢弃的情况（硬规则，优先于打分）

以下任一命中，**直接判 C，不用打分**：

1. **超短无信息**：去掉标点后 < 8 个汉字，且无场景（"爱了""泪目""yyds"）。
2. **灌水抢楼**："前排/沙发/板凳/999+/打卡/mark/顶" 等占楼词。
3. **纯表情 / 乱码 / 火星文**：表情符号或特殊符号占比过高，或含不可识别乱码。
4. **广告 / 引流**：含联系方式（V信/微信/电话/QQ）、"加V""刷赞""日入过千""代练""出二手"等。
5. **无关跑题**：内容与音乐 / 情绪无关（聊吃饭、问路、卖房、闲杂对话）。
6. **重复内容**：与已入库样本文本重复或高度雷同（复制粘贴的爆款评论只保留 1 条）。
7. **敏感 / 违规**：涉政、辱骂、色情、人身攻击等 —— 一律丢弃。

## 3. 标注产出的标准格式

每条**保留**的数据，最终整理成大模型 SFT 的标准对话格式：

```json
{
  "messages": [
    {"role": "system",    "content": "你是一个擅长写网易云风格乐评的助手，你的文字富有画面感和故事性，能触动人心。"},
    {"role": "user",      "content": "请为歌曲《歌名》（歌手）写一段乐评，背景是<情绪背景>。"},
    {"role": "assistant", "content": "<清洗/改写后的乐评正文>"}
  ]
}
```

字段要求：
- **system**：固定人设，统一用上面这句，保证风格一致。
- **user**：由标注员根据乐评内容**反推**出的「歌曲 + 情绪背景」提问（背景用 2~4 个关键词，如"深夜、孤独、怀念"）。
- **assistant**：核心训练目标，必须是**清洗 + 达标后的乐评正文**，40~120 字为宜。

---

## 4. 文本清洗细则（改写 B 级 / 规整 A 级时执行）

1. **去首尾空白、全角空格、多余换行**。
2. **繁体转简体**（"這首歌真美" → "这首歌真美"）。
3. **去重复标点**（"，，" → "，"；"。。。" → "…"）。
4. **去表情堆砌**（保留至多 1 个有意义的表情，其余删除）。
5. **纠正明显错别字 / 火星文**（"呮媞想埘暒" → "只是想起某些"）。
6. **不改变原意、不无中生有**：改写只做"润色成型"，不能编造原文没有的情节。

---

## 5. 标注一致性与质检

- **双人抽检**：每 100 条随机抽 10 条由第二人复核，两人评级不一致率需 < 15%。
- **边界样本讨论**：5~7 分的 B 级样本是争议高发区，需在标注例会上对齐标准。
- **及时反馈**：发现指南未覆盖的新情况（如新型广告话术），记录并反馈，迭代本规范。
- 质检脚本见 `quality_check.py`，自动检查：字段完整性、长度分布、重复率、疑似漏网噪声。
